🗺️ Roadmap 🚀 Before launch 🇩🇪 Germany/EU ready

План развития сайта Tatsiana Rukhlenko

Наглядная версия roadmap: что нужно сделать до публичного запуска, что влияет на заявки, legal, аналитику, SEO и дальнейший рост сайта.

🔐 Legal first

Сначала закрываем Impressum, Datenschutz, адрес, VAT и квалификации.

📅 Booking next

Потом решаем: Calendly, form service или backend для заявок.

📊 Analytics later

Cloudflare Web Analytics можно добавить как бесплатный privacy-friendly вариант.

🌱 Growth after launch

Отзывы, услуги, блог и SEO-контент добавляем после базового запуска.

1

Перед публичным запуском

Самое важное для доверия, законности и готовности сайта.

🔴 Critical
🏠
1.1

Заменить Adresse folgt

P1

Добавить реальный рабочий адрес, который нужен для немецкого Impressum.

🧾
1.2

Уточнить VAT ID

P1

Заменить USt-IdNr.: folgt только после подтверждения настоящего VAT ID. Не использовать временное значение 123456789.

🎓
1.3

Добавить квалификации

P1

Добавить проверенные данные о квалификациях, сертификатах и обучении Tatsiana.

⚖️
1.4

Проверить legal-тексты

P1

Финально проверить Impressum и Datenschutzerklärung у квалифицированного немецкого legal/privacy специалиста.

📅
1.5

Добавить реальную Calendly-ссылку

P2

Добавить ссылку, когда она будет готова. До этого не показывать Calendly в публичном тексте сайта.

2

Запись на консультацию и формы

Как человек оставляет заявку и что происходит с его данными.

🎯 Conversion
🧭
2.1

Выбрать формат Calendly

P2

Решить, будет ли Calendly открываться внешней ссылкой или встраиваться прямо на страницу.

🔐
2.2

Обновить Datenschutz при embed

P1

Если Calendly будет встроен на страницу, обновить Datenschutzerklärung, потому что будет загружаться сторонний виджет.

📨
2.3

Заменить mailto:

P2

Рассмотреть полноценный form service или backend endpoint вместо текущего email fallback.

🗄️
2.4

Документировать хранение данных

P2

Если используется form service, указать, где хранятся данные формы и как долго они сохраняются.

3

Аналитика и согласие пользователя

Как измерять заявки и не создавать лишних privacy-рисков.

📊 Analytics
🧘
3.1

MVP без tracking

Info

В MVP сейчас намеренно нет аналитики и трекинга.

🟠
3.2.1

Cloudflare Web Analytics

P2

Бесплатный privacy-friendly вариант аналитики.

📈
3.2.2

Plausible, Fathom или Vercel Analytics

Option

Privacy-friendly варианты для понимания трафика и заявок.

🎯
3.2.3

Google Analytics или Meta Pixel

Later

Использовать только если понадобятся реклама, remarketing или более сложный marketing tracking.

💻
3.3

Cloudflare вариант 1

P2

Если сайт не размещён через Cloudflare Pages, вручную добавить Cloudflare JavaScript snippet перед </body> в index.html и impressum.html после получения реального data-cf-beacon token.

🇩🇪
3.4

Добавить текст в Datenschutzerklärung

P1

Если добавляется Cloudflare Web Analytics, включить этот раздел и юридически проверить его перед запуском:

Wir verwenden Cloudflare Web Analytics zur anonymisierten Auswertung der Seitennutzung. Cloudflare Web Analytics verwendet keine Cookies und erhebt nach Angaben von Cloudflare keine personenbezogenen Daten der Besucher.

✅
3.5

Consent до загрузки инструментов

P1

Если добавляются analytics, pixels, maps, videos или embedded widgets, реализовать consent до загрузки этих инструментов.

⚙️
3.6

Cookie/settings control

P2

Добавить управление настройками, чтобы пользователь мог позже изменить своё согласие.
